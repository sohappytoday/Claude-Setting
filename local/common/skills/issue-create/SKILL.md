# Issue Create

## 순서
1. 작업 내용을 듣고 적절한 템플릿 선택: Bug / Chore / Feature / Refactor / Test
2. @templates/{선택한템플릿}.md 참고해서 이슈 내용 작성
3. title, labels, Assignee 확인
4. MCP(GitHub) 또는 사용자에게 내용 전달
5. 이슈 번호 확인 후 브랜치명 안내: feature/{issue_number}-{short-description}

## 규칙
- PR 전에 반드시 이슈 먼저 생성
- 이슈 번호 없으면 PR 불가
- Assignee, Label 반드시 선택
